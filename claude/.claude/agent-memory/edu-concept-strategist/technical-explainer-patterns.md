# Technical Explainer Video Patterns

## Narrative Arc Template (14-18 min videos)
1. **Hook** (60-90s): Prediction-and-reveal or surprising demo
2. **Foundation** (3-4 min): Define the core concept, establish why it exists
3. **Mechanism** (4-5 min): How it works under the hood -- densest segment, slow pacing
4. **Exploration** (3-4 min): Surprising behaviors, edge cases -- re-energizes after dense segment
5. **Implications** (4-5 min): "Why you should care" -- connect to money, time, correctness
6. **Close** (60s): 3-takeaway recap + call to action

## Segment Ordering Principle
- Place the densest/most abstract segment (e.g., algorithm explanation) in position 3, flanked by concrete/example-driven segments on both sides. This prevents cognitive overload fatigue.

## Visual Priorities for Technical Topics
1. Color-coded overlays on real data (highest impact, use throughout)
2. Step-by-step animations for iterative processes
3. Side-by-side comparisons for "same input, different system" demonstrations
4. Bar/budget diagrams for resource constraint explanations

## Duration Calibration
- Developer audiences tolerate up to 18 minutes if pacing varies and content is dense.
- Below 10 minutes often means cutting the "mechanism" segment, which undermines depth.
- Sweet spot for single-concept technical explainers: 14-16 minutes.
